# ເອກະສານອະທິບາຍຖານຂໍ້ມູນ (Database Schema Documentation - Complete Version)
ລະບົບບໍລິຫານ ບຸກຄະລາກອນ ແລະ ເງີນເດືອນກອງພົນທີ 3

ເອກະສານສະບັບນີ້ໄດ້ລວບລວມ ແລະ ອະທິບາຍທຸກໆ **16 ຕາຕະລາງ (Tables)** ທີ່ມີຢູ່ໃນຖານຂໍ້ມູນ `project_police2025` ຢ່າງລະອຽດ ເພື່ອກຽມຄວາມພ້ອມໃນການຕອບຄຳຖາມອາຈານ.

---

## 1. ໂຄງສ້າງການເຊື່ອມໂຍງລວມ (Database Architecture)
ຖານຂໍ້ມູນນີ້ຖືກອອກແບບເປັນ **Relational Database** ໂດຍແບ່ງກຸ່ມຕາຕະລາງອອກເປັນ 5 ກຸ່ມຫຼັກ:
1. **ກຸ່ມພູມສາດ/ທີ່ຢູ່**: `province`, `distict`, `village`
2. **ກຸ່ມໂຄງສ້າງການຈັດຕັ້ງ**: `department`, `office`, `panak`, `units`
3. **ກຸ່ມຕຳແໜ່ງ ແລະ ຊັ້ນ**: `positions`, `positions_level`, `rank_position`
4. **ກຸ່ມຂໍ້ມູນຫຼັກພະນັກງານ**: `users`, `officers`, `educations`
5. **ກຸ່ມທຸລະກຳ ແລະ ການເງິນ**: `level_up`, `transfer_records`, `salaries`

## 2. ລາຍລະອຽດທັງໝົດ 16 ຕາຕະລາງ (All 16 Tables Specification)

### 2.1 ຕາຕະລາງຜູ້ໃຊ້ລະບົບ (`users`)
* **ໜ້າທີ່**: ເກັບບັນຊີຜູ້ໃຊ້ງານລະບົບ (ແອັດມິນ ແລະ ພະນັກງານ) ເພື່ອເຂົ້າສູ່ລະບົບ.
* **ຟິລທີ່ສຳຄັນ**:
  * `user_id` (PK): ລະຫັດຜູ້ໃຊ້.
  * `username` / `password`: ຊື່ ແລະ ລະຫັດຜ່ານ (ເຂົ້າລະຫັດ SHA-512).
  * `role`: ສິດທິໃນລະບົບ (admin, user).
  * `img`: ຮູບໂປຣຟາຍຂອງຜູ້ໃຊ້.

### 2.2 ຕາຕະລາງປະຫວັດພະນັກງານ/ນາຍທະຫານ (`officers`)
* **ໜ້າທີ່**: ຕາຕະລາງຫຼັກທີ່ເກັບຂໍ້ມູນປະຫວັດສ່ວນຕົວ, ບ່ອນເກີດ, ທີ່ຢູ່ປັດຈຸບັນ, ຂໍ້ມູນຄອບຄົວ, ວດປ ເຂົ້າພັກ/ຊາວໜຸ່ມ/ແມ່ຍິງ, ໄຟລ໌ເອກະສານ ແລະ ຮູບພາບ.
* **ຟິລທີ່ສຳຄັນ**:
  * `officer_id` (PK): ລະຫັດພະນັກງານ.
  * `d_id`, `o_id`, `pk_id`, `u_id` (FK): ເຊື່ອມໂຍງສັງກັດ (ກົມກອງ, ຫ້ອງ, ພະແນກ, ໜ່ວຍງານ).
  * `l_id`, `pt_id` (FK): ເຊື່ອມໂຍງຊັ້ນ ແລະ ຕຳແໜ່ງ.
  * `pro_id`, `dis_id`, `v_id` (FK): ເຊື່ອມໂຍງທີ່ຢູ່ບ່ອນເກີດ.
  * `photo_img`: ພາຣາມິເຕີເກັບຮູບພາບພະນັກງານ.
  * `file_document`: ໄຟລ໌ເອກະສານຄັດຕິດ (PDF/Word).

### 2.3 ຕາຕະລາງກົມກອງຂຶ້ນກັບ (`department`)
* **ໜ້າທີ່**: ເກັບລາຍຊື່ກົມກອງໃຫຍ່ທີ່ຂຶ້ນກັບກອງພົນທີ 3.
* **ຟິລທີ່ສຳຄັນ**: `d_id` (PK), `d_name` (ຊື່ກົມກອງ).

### 2.4 ຕາຕະລາງກົມ/ຫ້ອງ (`office`)
* **ໜ້າທີ່**: ເກັບລາຍຊື່ຫ້ອງການຍ່ອຍທີ່ຂຶ້ນກັບແຕ່ລະກົມກອງ.
* **ຟິລທີ່ສຳຄັນ**: `o_id` (PK), `d_id` (FK -> ເຊື່ອມໄປຫາກົມກອງ), `o_name` (ຊື່ຫ້ອງການ).

### 2.5 ຕາຕະລາງພະແນກ (`panak`)
* **ໜ້າທີ່**: ເກັບລາຍຊື່ພະແນກທີ່ຂຶ້ນກັບແຕ່ລະຫ້ອງການ.
* **ຟິລທີ່ສຳຄັນ**: `pk_id` (PK), `o_id` (FK -> ເຊື່ອມໄປຫາຫ້ອງການ), `pk_name` (ຊື່ພະແນກ).

### 2.6 ຕາຕະລາງໜ່ວຍງານ (`units`)
* **ໜ້າທີ່**: ເກັບລາຍຊື່ໜ່ວຍງານນ້ອຍສຸດທີ່ຂຶ້ນກັບແຕ່ລະພະແນກ.
* **ຟິລທີ່ສຳຄັນ**: `u_id` (PK), `pk_id` (FK -> ເຊື່ອມໄປຫາພະແນກ), `u_name` (ຊື່ໜ່ວຍງານ).

### 2.7 ຕາຕະລາງຊັ້ນ (`positions_level`)
* **ໜ້າທີ່**: ເກັບຂໍ້ມູນຊັ້ນຍົດຂອງທະຫານ (ເຊັ່ນ: ສິບເອກ, ຮ້ອຍໂທ, ພັນຕີ...).
* **ຟິລທີ່ສຳຄັນ**: `l_id` (PK), `l_name` (ຊື່ຊັ້ນຍົດ), `l_img` (ຮູບພາບເຄື່ອງໝາຍຊັ້ນຍົດ).

### 2.8 ຕາຕະລາງໜ້າທີ່ຮັບຜິດຊອບ/ຕຳແໜ່ງ (`positions`)
* **ໜ້າທີ່**: ເກັບລາຍຊື່ຕຳແໜ່ງ ຫຼື ໜ້າທີ່ຮັບຜິດຊອບໃນວຽກງານ.
* **ຟິລທີ່ສຳຄັນ**: `pt_id` (PK), `pt_name` (ຊື່ຕຳແໜ່ງ).

### 2.9 ຕາຕະລາງກຳນົດປີເລື່ອນຊັ້ນ (`rank_position`)
* **ໜ້າທີ່**: ເກັບຂໍ້ມູນເກນກຳນົດໄລຍະເວລາ (ປີ ແລະ ເດືອນ) ທີ່ຕ້ອງຄົບກຳນົດໃນການເລື່ອນຊັ້ນແຕ່ລະຂັ້ນ.
* **ຟິລທີ່ສຳຄັນ**: `r_id` (PK), `l_id` (FK -> ເຊື່ອມໂຍງກັບຊັ້ນ), `r_years` (ຈຳນວນປີກຳນົດ), `r_month` (ຈຳນວນເດືອນກຳນົດ).

### 2.10 ຕາຕະລາງການເລື່ອນຊັ້ນ (`level_up`)
* **ໜ້າທີ່**: ບັນທຶກປະຫວັດການເລື່ອນຊັ້ນຍົດຕົວຈິງຂອງທະຫານແຕ່ລະຄົນ.
* **ຟິລທີ່ສຳຄັນ**: `level_id` (PK), `officer_id` (FK), `l_id` (FK -> ຊັ້ນຍົດໃໝ່), `level_date` (ວັນທີເລື່ອນຊັ້ນ).

### 2.11 ຕາຕະລາງຍ້າຍເຂົ້າ-ຍ້າຍອອກ (`transfer_records`)
* **ໜ້າທີ່**: ເກັບປະຫວັດການໂຍກຍ້າຍກົມກອງ/ໜ່ວຍງານຂອງກຳລັງພົນ.
* **ຟິລທີ່ສຳຄັນ**: `tran_id` (PK), `officer_id` (FK), `transfer_tyep` (ປະເພດ: IN = ຍ້າຍເຂົ້າ, OUT = ຍ້າຍອອກ), `transfer_date` (ວັນທີຍ້າຍ).

### 2.12 ຕາຕະລາງປະຫວັດການສຶກສາ (`educations`)
* **ໜ້າທີ່**: ບັນທຶກລະດັບການສຶກສາ ແລະ ວິຊາສະເພາະຂອງພະນັກງານແຕ່ລະຄົນ.
* **ຟິລທີ່ສຳຄັນ**: `ed_id` (PK), `officer_id` (FK), `name_level` (ລະດັບການສຶກສາ), `kanangvixah` (ຂະແໜງຮຽນ).

### 2.13 ຕາຕະລາງເງິນເດືອນ (`salaries`)
* **ໜ້າທີ່**: ຄິດໄລ່ ແລະ ບັນທຶກລາຍລະອຽດເງິນເດືອນຂອງທະຫານ/ລັດຖະກອນໃນແຕ່ລະເດືອນ.
* **ຟິລທີ່ສຳຄັນ**:
  * `salary_id` (PK): ລະຫັດລາຍການເງິນເດືອນ.
  * `officer_id` (FK): ເຊື່ອມຫາພະນັກງານ.
  * `salary_type`: ປະເພດ (`soldier` = ທະຫານ, `employee` = ພະນັກງານລັດຖະກອນ).
  * `base_salary`: ເງິນເດືອນພື້ນຖານ.
  * `salary_increase_15`: ເງິນອຸດໜູນເພີ່ມ 15%.
  * `allowance`: ເງິນອຸດໜູນຕຳແໜ່ງ/ຊັ້ນຍົດ.
  * `deduct_tax` / `deduct_phone`: ລາຍການຫັກພາສີ ຫຼື ຄ່າໂທລະສັບ.

### 2.14 ຕາຕະລາງແຂວງ (`province`)
* **ໜ້າທີ່**: ເກັບລາຍຊື່ແຂວງທັງໝົດໃນປະເທດລາວ.
* **ຟິລທີ່ສຳຄັນ**: `pro_id` (PK), `pro_name` (ຊື່ແຂວງ).

### 2.15 ຕາຕະລາງເມືອງ (`distict`)
* **ໜ້າທີ່**: ເກັບລາຍຊື່ເມືອງທັງໝົດ.
* **ຟິລທີ່ສຳຄັນ**: `dis_id` (PK), `pro_id` (FK -> ເຊື່ອມຫາແຂວງ), `dis_name` (ຊື່ເມືອງ).

### 2.16 ຕາຕະລາງບ້ານ (`village`)
* **ໜ້າທີ່**: ເກັບລາຍຊື່ບ້ານທັງໝົດ.
* **ຟິລທີ່ສຳຄັນ**: `v_id` (PK), `dis_id` (FK -> ເຊື່ອມຫາເມືອງ), `pro_id` (FK -> ເຊື່ອມຫາແຂວງ), `v_name` (ຊື່ບ້ານ).

## 3. ແຜນວາດຄວາມສຳພັນລະຫວ່າງຕາຕະລາງ (ER Relationship Logic)

```
[province] (1) 
    └── [distict] (1) 
           └── [village] (1)
                  └── [officers] (Many) ── (1) [department] -> (1) [office] -> (1) [panak] -> (1) [units]
                          │
                          ├── (Many) [level_up] (ບັນທຶກປະຫວັດເລື່ອນຊັ້ນ)
                          ├── (Many) [transfer_records] (ບັນທຶກປະຫວັດໂຍກຍ້າຍ)
                          ├── (Many) [educations] (ບັນທຶກປະຫວັດການສຶກສາ)
                          └── (Many) [salaries] (ບັນທຶກການຈ່າຍເງິນເດືອນ)
```

## 4. ຄຳຖາມຫຼັກທີ່ອາຈານມັກຖາມ (Top Questions for Project Defense)

### ຄຳຖາມ 1: ຖານຂໍ້ມູນນີ້ມີລະບົບເງິນເດືອນ (`salaries`) ແຍກປະເພດແນວໃດ?
* **ຄຳຕອບ**: ລະບົບມີຟິລ `salary_type` ເປັນ `ENUM('soldier', 'employee')` ເພື່ອແຍກລະຫວ່າງ "ນາຍ ແລະ ພົນທະຫານ" ແລະ "ພະນັກງານລັດຖະກອນ" ເນື່ອງຈາກທັງສອງປະເພດນີ້ມີເກນເງິນເດືອນພື້ນຖານ ແລະ ຄ່າອຸດໜູນນະໂຍບາຍທີ່ແຕກຕ່າງກັນ.

### ຄຳຖາມ 2: ເຮົາຈະຮູ້ໄດ້ແນວໃດວ່າທະຫານຄົນໃດຄົບກຳນົດເລື່ອນຊັ້ນແລ້ວ?
* **ຄຳຕອບ**: ລະບົບຈະນຳໃຊ້ຂໍ້ມູນຈາກ 3 ຕາຕະລາງຮ່ວມກັນຄື:
  1. ເບິ່ງວັນທີເລື່ອນຊັ້ນຫຼ້າສຸດຈາກ `officers.date_level` (ຫຼື ຈາກ `level_up`).
  2. ປຽບທຽບກັບເກນປີກຳນົດໃນຕາຕະລາງ `rank_position` ຂອງຊັ້ນຍົດນັ້ນໆ.
  3. ນຳໃຊ້ມາຄິດໄລ່ປຽບທຽບກັບເວລາປັດຈຸບັນ ເພື່ອແຈ້ງເຕືອນລາຍຊື່ຜູ້ຄົບກຳນົດເລື່ອນຊັ້ນຍົດ.

### ຄຳຖາມ 3: ການເກັບຮູບພາບພະນັກງານໃນ `officers.photo_img` ແລະ `users.img` ເກັບແນວໃດ?
